# Root bun check

## Goal

Get root `bun check` green by fixing the real failing seam, not by muting the
check.

## Plan

1. Reproduce the current `bun check` failure and isolate the blocking step.
2. Find any relevant learnings for the failing area.
3. Add or update the right test seam if the failure is in maintained package
   behavior.
4. Apply the minimal fix.
5. Re-run `bun check` and the direct failing lanes.

## Findings

- `bun check` was blocked in `scenario:check -- convex-next-all`, not in the
  root static gates.
- Local `kitcn codegen` on auth-enabled Convex apps could reach auth
  config analysis before `convex/.env` had been pushed into the local
  deployment.
- That made scenario apps fail with auth env errors like missing `DEPLOY_ENV`
  even though the value existed on disk in `convex/.env`.
- The right seam was package codegen, not the scenario harness.
- Fresh `bun check` passed after teaching local auth-aware codegen to do a
  prepare/complete env sync around the codegen pass and avoiding duplicate sync
  in init-owned bootstrap flows.
- `fixtures:sync` should refresh committed dependency pins before generating
  snapshots, but it cannot call the full validated `deps:sync` command because
  that command already calls `fixtures:sync`.
