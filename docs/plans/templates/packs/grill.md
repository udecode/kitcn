# grill pack

Use when an expensive product, API, architecture, or DX direction needs a
vision-backed pressure test.

Start Gates:
| Gate | Applies | Evidence |
| --- | --- | --- |
| Vision and named source read | pending | pending |
| Exact public/runtime owners inspected | pending | pending |
| Core irreversible choices listed | pending | pending |

Work Checklist:
- [ ] Grill pack: user/job, kitcn ownership, familiar mental model, public API,
      types, auth/data flow, bundles, CLI/generated ownership, proof, and cut
      scope are challenged.
- [ ] Grill pack: contradictions force a decision rather than an abstraction.
- [ ] Grill pack: rejected options and disconfirming evidence are recorded.
- [ ] Grill pack: durable findings are written to the active plan or a bounded
      brainstorm owner.

Completion Gates:
| Gate | Applies | Required action | Evidence |
| --- | --- | --- | --- |
| Owner/decision map | yes | Settle core irreversible decisions | pending |
| Contradiction pass | yes | Resolve or block each contradiction | pending |
| Next owner | yes | Route to milestone, PRD, major-task, task, or stop | pending |
