Plugins are first-class in Better Convex. Register once in `defineSchema`, then scaffold runtime files with `better-convex add`.

## Plugin contract

`OrmSchemaPlugin` is schema-only:

1. `key`
2. `schema.tableNames`
3. `schema.inject(...)`
4. optional `schema.relations(...)`

```ts showLineNumbers
type OrmSchemaPlugin = {
  key: string;
  schema: {
    tableNames: readonly string[];
    inject: <TSchema extends Record<string, unknown>>(
      schema: TSchema
    ) => TSchema & Record<string, unknown>;
    relations?: (helpers: any) => Record<string, unknown>;
  };
};
```

CLI presets/templates live in the first-party catalog (`packages/better-convex/src/cli/plugin-catalog.ts`), not in schema plugin objects.

Plugin relation composition:

- Plugin `schema.relations(...)` outputs are merged with app `defineSchema(..., { relations })`.
- Merge order is plugin order, then app relations.
- Duplicate relation field keys (`table.field`) throw.

## Discovery model

`defineSchema({ plugins })` is the source of truth for plugin discovery:

```ts showLineNumbers {2,6}
import { defineSchema } from 'better-convex/orm';
import { resendPlugin } from '@better-convex/resend/schema';

export default defineSchema(tables, {
  plugins: [resendPlugin()],
});
```

First-party plugin schema factories can also expose table overrides for advanced setups:

```ts showLineNumbers
import { convexTable, text } from 'better-convex/orm';
import { resendPlugin } from '@better-convex/resend/schema';

const customResendEmails = convexTable('resend_emails', {
  custom: text().notNull(),
});

export default defineSchema(tables, {
  plugins: [
    resendPlugin({
      tables: {
        resendEmails: customResendEmails,
      },
    }),
  ],
});
```

Codegen behavior:

- Installed plugin keys are used for generated typing/runtime context.
- Plugin runtime modules are scaffold-owned; `codegen` does not emit plugin runtime files.
- Stale `generated/plugins/**` artifacts are removed automatically.

## Ownership model (shadcn-style parity)

- Generated internals are framework-owned and overwrite-safe.
- User scaffold files are user-owned.
- Plugin function module default: `<functionsDir>/plugins/<plugin>.ts`.
- Plugin helper modules default: `convex/<paths.lib>/plugins/<plugin>/...`.
- `better-convex add <plugin>` creates missing scaffold files.
- `better-convex diff <plugin>` previews scaffold drift (read-only).
- Plugin inventory/state is tracked in `<functionsDir>/plugins.lock.json`.

## Available plugins

| Plugin | What it gives you |
| --- | --- |
| [`ratelimit`](/docs/plugins/ratelimit) | Convex-native rate limiting with middleware + React hook support |

## Built-in internal plugins

These are always active and do not require explicit registration:

| Plugin | Purpose |
| --- | --- |
| `aggregatePlugin` | Aggregate storage + runtime support |
| `migrationPlugin` | Migration state + runtime support |
