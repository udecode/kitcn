Reference for the files that `better-convex init -t next --yes` writes.

Use the CLI first:

```bash showLineNumbers
npx better-convex init
npx better-convex init -t next --yes
npx better-convex add resend
npx better-convex add ratelimit
```

This page is not a second setup flow. It is the file-level reference for the scaffold the CLI already emits.

## Ownership

`init -t next` has two sources of truth:

- shadcn owns the app shell and client utility files:
  - `app/layout.tsx`
  - `app/page.tsx`
  - `app/globals.css`
  - `components/theme-provider.tsx`
  - `lib/utils.ts`
  - `components.json`
  - `eslint.config.mjs`
  - `next.config.mjs`
  - `postcss.config.mjs`
- Better Convex owns the integration layer:
  - `components/providers.tsx`
  - `lib/convex/*`
  - `.env.local`
  - `convex/*`
  - `concave.json`
  - the package.json baseline patch
  - the `@convex/*` tsconfig alias patch
  - the minimal `layout.tsx` patch that mounts `Providers`

If the target app already uses `src/`, the same scaffold is written under `src/app`, `src/components`, and `src/lib`. Mixed `src` + root layouts fail fast instead of guessing.

## Root Config

### package.json

The scaffold keeps the standard Next scripts and adds the Better Convex baseline dependency patch.

```json title="package.json" showLineNumbers
{
  "scripts": {
    "dev": "next dev --turbopack",
    "codegen": "better-convex codegen",
    "build": "next build",
    "start": "next start",
    "lint": "eslint",
    "format": "prettier --write \"**/*.{ts,tsx}\"",
    "typecheck": "tsc --noEmit"
  }
}
```

If `codegen` is already taken, the scaffold adds `convex:codegen` instead.

### tsconfig.json

Better Convex patches the existing tsconfig instead of replacing it.

```json title="tsconfig.json" showLineNumbers {5}
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"],
      "@convex/*": ["./convex/shared/*"]
    }
  }
}
```

`@/*` stays on the existing root (`./*` or `./src/*`). `@convex/*` is added for generated shared API imports.

### components.json

Better Convex preserves the shadcn config and only adjusts `tailwind.css` when the resolved app root requires it.

```json title="components.json" showLineNumbers {7}
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "base-nova",
  "tailwind": {
    "config": "",
    "baseColor": "neutral",
    "cssVariables": true,
    "css": "app/globals.css"
  }
}
```

## App Integration

### app/layout.tsx

Better Convex does not replace the shadcn layout. It patches the existing `children` slot so the client core is mounted through `Providers`.

```tsx title="app/layout.tsx" showLineNumbers
import { Providers } from '@/components/providers';

// ...existing shadcn layout

<ThemeProvider>
  <Providers>{children}</Providers>
</ThemeProvider>
```

If the target layout is too custom to patch safely, init should fail instead of guessing.

### components/providers.tsx

```tsx title="components/providers.tsx" showLineNumbers
import type { ReactNode } from 'react';

import { BetterConvexProvider } from '@/lib/convex/convex-provider';
import { HydrateClient } from '@/lib/convex/rsc';

export function Providers({ children }: { children: ReactNode }) {
  return (
    <BetterConvexProvider>
      <HydrateClient>{children}</HydrateClient>
    </BetterConvexProvider>
  );
}
```

## Better Convex Baseline

### convex/functions/schema.ts

```ts title="convex/functions/schema.ts" showLineNumbers
import { defineSchema } from 'better-convex/orm';

export const tables = {};

export default defineSchema(tables);
```

### convex/lib/crpc.ts

```ts title="convex/lib/crpc.ts" showLineNumbers
import { initCRPC } from '../functions/generated/server';

const c = initCRPC
  .meta<{
    auth?: 'optional' | 'required';
    role?: 'admin';
    ratelimit?: string;
  }>()
  .create();

export const publicQuery = c.query;
export const publicAction = c.action;
export const publicMutation = c.mutation;

export const privateQuery = c.query.internal();
export const privateMutation = c.mutation.internal();
export const privateAction = c.action.internal();

export const publicRoute = c.httpAction;
export const router = c.router;
```

## Notes

- `templates/next` is a repo fixture, not the public contract.
- The fixture is normalized for repo-local development only (`better-convex` rewritten to `workspace:*`, repo-safe package name, stripped volatile files).
- The fixture comes from real `better-convex init -t next` output plus repo-only normalization. `check:templates` builds and packs the local `better-convex` package, validates a fresh generated app with install, lint, typecheck, and build, then compares the normalized snapshot. It fails if template init ever falls back to a stubbed runtime.
- If you want the scaffold, use `better-convex init -t next`. Do not cargo-cult the fixture.
